## 5. CI/CD Pipelines

### 5 Workflows GitHub Actions

| Workflow | Trigger | Steps |
|----------|---------|-------|
| **Backend CI** | Push/PR on `app/**`, `tests/**`, `alembic/**`, `pyproject.toml` | Ruff → mypy → pytest + coverage → Codecov (flag: backend) → Bandit |
| **Frontend CI** | Push/PR on `frontend/**` | ESLint → tsc → Vitest + coverage → Codecov (flag: frontend) → Build |
| **Agent CI** | Push/PR on `agent/**` | Ruff → mypy → pytest + coverage → Codecov (flag: agent) |
| **Docker Build & Push** | Push to `main` (ignores `.md`, `.opencode/`) | Build & push 3 images to `ghcr.io/<owner>/beacon-<component>:latest` + `:sha` |
| **E2E Tests** | Push to `main` + daily at 06:00 UTC | `docker compose up -d --build` → wait health → `npx playwright test` → upload report → auto-issue on failure |

### Coverage (Codecov)

- **Config:** `codecov.yml` na raiz do projeto
- **Target:** 80% per flag (backend, frontend, agent)
- **Threshold:** 2% de tolerância
- **Patch target:** 80% (código novo/alterado)
- **Carryforward:** Habilitado (CI não falha para componentes sem alteração)
